## Product Selected

**Online Food Delivery Web Application**

## Introduction

In today's fast-paced world, online food delivery services have become an essential part of daily life. Customers prefer ordering food online because it saves time and provides convenience. The Online Food Delivery Web Application is designed to offer users a simple, efficient, and user-friendly platform for browsing restaurants, selecting food items, placing orders, and making secure payments.

## Problem Statement

Traditional food ordering methods are time-consuming and may create inconvenience for customers. Users require a digital platform that allows them to explore restaurants, order food, and complete payments quickly and securely. Therefore, a web-based food delivery application is proposed to improve the overall ordering experience.

## Objective

The main objective of this project is to design a prototype of an Online Food Delivery Web Application using the Figma tool. The application aims to provide an easy-to-use interface that simplifies the process of food ordering and enhances customer satisfaction.

## Prototype Tool Used

**Figma**

Figma was used to create the user interface and interactive prototype. The prototype demonstrates how users navigate through different screens and complete the food ordering process.

## Description

The Online Food Delivery Web Application enables customers to browse restaurants, view food items, add products to the cart, choose a payment method, and place orders successfully. The prototype focuses on providing a smooth and efficient user experience. Different screens are connected using Figma's prototype feature to simulate the actual workflow of the application.

## Features

* User-friendly interface.
* Secure login system.
* Search and browse food items.
* Restaurant listing with menu details.
* Add items to cart functionality.
* View total amount and order summary.
* Multiple payment options.
* Order confirmation and tracking facility.
* Fast and convenient food ordering process.

## Screens Designed

### 1. Home Page

The home page acts as the main dashboard of the application. It contains:

* Logo and navigation bar.
* Search bar for finding food items.
* Food categories such as Pizza, Burger, Drinks, and Desserts.
* Popular dishes section.
* Order Now button.

### 2. Login Page

The login page allows users to access their accounts. It includes:

* Email address field.
* Password field.
* Login button.
* Sign Up option for new users.

### 3. Restaurant Page

This page displays available restaurants and menu items. It contains:

* Restaurant names and images.
* Food item descriptions.
* Price details.
* Add to Cart button.

### 4. Cart Page

The cart page provides an order summary. It contains:

* Selected food items.
* Quantity details.
* Total price calculation.
* Checkout button.

### 5. Payment Page

This page allows users to make payments securely. It includes:

* UPI payment option.
* Debit/Credit card payment.
* Cash on Delivery option.
* Confirm Payment button.

### 6. Order Confirmation Page

This page confirms the successful placement of the order. It contains:

* Order confirmation message.
* Order ID details.
* Estimated delivery information.
* Track Order button.

## User Flow

The application follows the following workflow:

**Login Page → Home Page → Restaurant Page → Cart Page → Payment Page → Order Confirmation Page**

This sequence ensures that users can easily navigate through the system and complete their orders without difficulty.
